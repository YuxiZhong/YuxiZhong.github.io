哈密顿图模型

1. 模型概览

1.1 学科分属

基本学科归属： 图论

需要背景知识： 哈密顿回路，动态规划，0-1整数规划，最小生成树

1.2 历史发展

1.2.1 哈密顿图的定义1

G=(V,E)是一个图，若G中一条通路通过每一个顶点一次且仅一次，称这条通路为哈密顿通路。若G中一个圈通过每一个顶点一次且仅一次，称这个圈为哈密顿圈。若一个图存在哈密尔顿圈，就称为哈密顿图。

判断哈密顿路径是NP完全问题，还没有有效的判断算法。

1.2.2 研究历史2

哈密尔顿（Hamilton）是爱尔兰的一位数学家和天文学家。他的一生是多姿多彩的，自从哈密尔顿发现“四元数”之后，他又发现了另外一种他命名为“The Icosian Calculus”的代数系统，这系统有加法和乘法的运算子（operators），可是乘法不满足交换律（Commutative law 即 xy=yx 这个规律）。

他发现这代数系统是和正则12面体（regularicosion polybedron）是有关系的。于是，哈密尔顿于1859年提出了所谓的“周游世界游戏”，即用一个正十二面体的20个顶点表示全世界的20个大城市（图1.1（a）），一个人要从某一个城市出发，经过每个城市恰好一次，然后回到出发点，问这样的旅行路线是否存在?这个游戏曾风靡一时。运用拓扑的思想，将这个正十二面体“拉平”就会得到一个与它同构的平面图（图1.1（b））。这样，这个游戏就转化为：要求沿着正十二面体的棱，怎样跑遍正则多面体上的所有顶点，而最后又能回到起点的问题。

图1.1：哈密顿提出的周游世界游戏

此后，由于这个游戏的缘故，人们就称这一类图为哈密尔顿图。其实哈密尔顿图的由来可以追溯到更早时期。在17-18世纪时，欧洲宫廷及一些贵族很喜欢玩西洋象棋，西洋象棋中的骑士（knight）是对应我们中国象棋的“马”，而它通常是刻成一个马头，跑法也是和中国象棋的“马”一样，走“日”线，即从日的一角沿着对角线跃到另一角

在1771年，就有一位名叫范德蒙（A. Vandermonde）的法国数学家，写了一篇文章研究所谓“棋盘的骑士问题”。这问题是这样：在8×8的棋盘格的随意一个位置上，放一个骑士，然后想法子使他跑遍棋盘的所有的格子，走过的不许再走，能不能使骑士最后回到原来的位置？

这个问题并不简单，许多象棋爱好者及数学家曾坐下来研究过这个问题。表1.1为一个情形的解，将棋盘的左下角的格选为起始位置，把它定为1，由表可见，骑士最后是能跑回1的。

表1.1：“棋盘骑士问题”的一个解法

18世纪的大数学家欧拉（Euler），在1759年曾系统地研究过这个问题，也得到了一些结果。以后德国数学家高斯也曾对这问题发生兴趣，花过一些时间研究它以及另外一个关于棋盘的“皇后问题”。

我们现在把棋盘上的格子对应在一个平面上的一个小圆点，这样我们在平面上就有64个小圆点。从一个格子出发用骑士的走法我们可以抵达最终不同数目的格子：如果是处在棋盘的四个角，只能有两种跑法；在其他边缘的格子就有三种跑法；一般当中的格子，就有四种肯的跑法。我们把平面上的点用弧线连接，两个点间存在一条弧线连接，当且仅当我们可以从他们对应的格子将骑士移动。该图经过所有的点一次且仅一次，这样的图就是众所周知的哈密尔顿图。

与欧拉图的情形不同，还未找到判断一个图是否是哈密顿图的非平凡的充要条件。事实上这是图论中尚未解决的主要问题之一。

美国图论数学家奥勒在1960年给出了一个图是哈密尔顿图的充分条件：对于顶点个数大于2的图，如果图中任意两点度的和大于或等于顶点总数，那这个图一定是哈密顿图。

哈密顿路径问题在上世纪七十年代初，终于被证明是“NP完备”的。据说具有这样性质的问题，难于找到一个有效的算法。实际上对于某些顶点数不到100的网络，利用现有最好的算法和计算机也需要比较荒唐的时间（比如几百年）才能确定其是否存在一条这样的路径。

一般地，寻找一个图的哈密顿圈问题是NP困难的。因此通常都是用穷举搜索的方法来判定一个图是否含有哈密顿路或圈。后来鲁宾（F.Rubin）利用推演的方法给出了一个好一点的搜索步骤来找出图里的一些或者全部的哈密顿路或圈，安格鲁因（D.Angluin）和瓦利安特（L.G.Valiant）设计的概率算法对哈密顿路或圈的寻找也是非常有用的。寻找哈密顿路的确定算法虽然很难有多项式时间的，但是这并不意味着只能进行时间复杂度为O(n!*n)暴力搜索。利用状态压缩动态规划，我们可以将时间复杂度降低到O(n²*n³)，具体算法是建立方程f[i][S][j]，表示经过了i个节点，节点都是集合S的，到达节点j时的最短路径。每次我们都按照点j所连的节点进行转移。

2. 模型介绍

2.1 具体模型介绍

2.1.1 哈密顿图的判别条件总结

  1. 充分条件：

    （1）狄拉克定理： 如果图G是一个具有n(n≥3)个顶点的简单图，并且图G中每个顶点的度数至少为n/2，那么图G是哈密顿图。

    （2） 奥勒定理： 设G是n(n≥3)阶无向简单图，如果G中任何一对不相邻的顶点度数之和都大于等于n，则G是哈密顿图。

    （4）在n(n≥2)阶有向图D中，如果所有有向边均用无向边代替，所得无向图中含生成子图Kn，则有向图中存在哈密顿图。

  2. 必要条件

    （1） 如果一个图的顶点可以用两种颜色染色，使相邻的顶点的颜色不同，那若结果染完两种颜色的顶点数量不一致，则一定不是哈密顿图。

    （2） 设无向图G=(V,E)是哈密顿图， V1 是V的任意的非空子集，则 p(G-V1)≤|V1| 。其中，p(G-V1)为从G中删除V1(删除V1中各顶点及关联的边)后所得的图的连通分支数。

2.1.2 哈密顿图的性质

  1. 哈密尔顿图一定是连通图。

    因为哈密尔顿回路一定是初级回路，因此也一定是简单回路，这就保证了图中点与点之间的连通性。

  2. 哈密尔顿回路的长度等于顶点的个数。

  3. 如果G=是一个哈密尔顿图，L是G中一条哈密尔顿回路，那么对G中任何一个顶点u而言，L上都有且只有两条边与顶点u相关联。当deg(u)=2时，与顶点u相关联的两条边都必在L上。当deg(v)>2时，与顶点u相关联的边中只能有两条边在L上。

  4. 哈密顿图一定无割点

2.1.3 寻找哈密顿回路的若干技巧

  1. 遇到度数为2的结点时，与该结点关联的两条边肯定是保留边

    如在图2.1的G1中，结点a，d，e，g为2度结点，由此可确定G11中7条保留边，用虚线表示，可以看出这7条保留边组成了哈密顿回路。

    图2.1：利用2度结点寻找哈密顿回路

  2. 巧用对称性，避免子回路
    子回路指的是由部分结点（非全部结点）组成的回路，在哈密尔顿回路中，子回路至少有一条边需要被去除，在去除某些边时，可根据图形的对称性，考虑其中一种情况即可。

3. 模型应用

3.1 常见应用场景1

3.1.1 ：旅行商问题

旅行商问题(Travelling salesman problem, 简称TSP)的定义为：给定一系列城市和每对城市之间的距离，求解访问每一座城市一次并回到起始城市的最短回路，即最短哈密顿回路。

实际问题中有很多问题都可以归结为旅行货郎问题。如：物资运输路线的设计；汽车应走怎样的路线使空车路线最短；工厂里在钢板上一些地方需要挖去小圆孔，自动焊割机割咀应该走怎样的路线，使空走的路线最短；城市里在一些地方铺设管道时，管道应该走怎样的路线，才能使管道的耗费最少等。

3.1.2 其他领域

由哈密顿图模型引出的旅行商问题有电缆和光缆布线、晶体结构分析、数据串聚类等多种用途。更重要的是．它提供了一个研究组合优化问题的理想平台。很多组合优化问题，比如背包问题、分配问题、车间调度问题都和TSP同属NP完全问题，它们都是同等难度的．如果其中一个能用多项式确定性算法解决，那么其他所有的NP完全问题也能用多项式算法解决。很多方法本来是从TSP发展起来的．后来推广到其他NP完全问题上去。

3.2 数学建模竞赛应用

CUMCM 1998B 最佳灾情巡视路线

3.2.1 问题复述

问题背景：1998年夏天，我国部分地区遭受水灾，为考察灾情、组织自救，某县领导决定，带领有关部门负责人到全县各乡（镇）、村巡视。巡视路线指从县政府所在地出发，走遍各乡（镇）、村，又回到县政府所在地。

全县乡镇、村的公路网示意图见图3.1，公路边的数字为该路段的距离，单位为km。

图3.1：全县公路网示意图

问题一：若分三组（路）巡视，试设计总路程最短且各组尽可能均衡的路线。

问题二：假定巡视人员在各乡（镇）停留时间$T=2小时$,在各村停留时间$t=1小时$,汽车行驶速度$V=35公里/小时$。要在24小时内完成巡视，至少应分几组；给出这种分组下最佳的巡视路线。

3.2.2 目标分析与模型转化

将公路网图中，每个乡（镇）或村看作图中的一个结点，各乡（镇）、村之间的公路看作图中对应结点的边，各条公路的长度（或行驶时间）看作对应边上的权，所给公路网就转化为加权网络图，问题就转化为在给定的加权网络图中寻找从定点$O$出发，行遍所有顶点至少一次再回到$O$点，使得总权（路程或时间）最小，我们的目标可以转化为找到使得总行程最短的旅行路径，与TSP问题的目标一致。

问题一是多个旅行商的最佳推销回路问题，即在加权图$G$中求顶点集$V$的划分$V_{1},V_{2},…,V_{n}$，将$G$分成$n$个生成子图$G\left [ V_{1} \right ],G\left [ V_{2} \right ],…,G\left [ V_{n} \right ]$，使得

  1. 顶点$O \in V_{i} , i=1,2,3,…,n $
  2. $\bigcup_{i=1}^{n} V_{i}=V(G)$
  3. $\frac{\operatorname{Max}_{i,j}\left|\omega\left(C_{i}\right)-\omega\left(C_{j}\right)\right|}{\operatorname{Max}_{i} \omega\left(C_{i}\right)} \leq \alpha$ ，其中$C_{i}$为$V_{i}$的导出子图G\left [V_{i} \right ]$中的最佳销售员回路，\omega \left(C_{i}\right)$为$C_{i}$的权，$i,j=1,2,3,…,n$
  4. $\sum_{i=1}^{n} \omega\left(C_{i}\right)=\operatorname{Min}$

定义：称$\alpha _ {0}=\frac{\operatorname{Max}_{i,j}\left|\omega\left(C_{i}\right)-\omega\left(C_{j}\right)\right|}{\operatorname{Max}_{i} \omega\left(C_{i}\right)}$为该分组的实际均衡度，$\alpha$为最大容许均衡度。

CUMCM 2013B 碎纸片的拼接复原
以下仅讨论问题一

3.2.1 问题复述

问题背景：破碎文件的拼接在司法物证复原、历史文献修复以及军事情报获取等领域都有着重要的应用。传统上，拼接复原工作需由人工完成，准确率较高，但效率很低。特别是当碎片数量巨大，人工拼接很难在短时间内完成任务。随着计算机技术的发展，人们试图开发碎纸片的自动拼接技术，以提高拼接复原效率。

对于给定的来自同一页印刷文字文件的碎纸机破碎纸片（仅纵切），建立碎纸片拼接复原模型和算法，并针对附件1、附件2给出的中、英文各一页文件的碎片数据进行拼接复原。如果复原过程需要人工干预，请写出干预方式及干预的时间节点。复原结果以表格形式表达。

3.2.2 目标分析与模型转化

目标分析
我们的目标是根据碎纸片间的匹配程度（这一匹配程度主要是由纸片边缘相似度决定的），找出使得所有碎纸片总的匹配程度最高的碎纸片排列方式。

模型转化
如果我们定义碎纸片间的“距离”为与碎纸片间的匹配程度反相关的量（两个碎纸片间匹配度越大，它们的距离越小），那么使得所有碎纸片两两间匹配程度的总和最高的碎纸片排列方式也就是使得相邻碎纸片的距离之和最小的碎纸片排列方式。
将每张碎纸片看成TSP问题中推销员经过的城市，碎纸片之间的“距离”看成城市间的距离，我们的目标可以转化为找到使得总行程最短的旅行路径，与TSP问题的目标一致。

3.2.3 问题的求解2

问题的求解过程可以分为以下步骤：

1. 将附近中给出的碎纸片图片进行数字化处理
2. 提取图片数据特征
3. 定义碎纸片匹配度函数及距离函数

5. 将碎纸片的距离作为参数投入TSP问题求解模型中，解得碎纸片的排序结果

3.2.3.1 图片数字化

对每张碎纸片图片进行以下二值化 处理：将每个像素点数字化为0或1。若像素点颜色为黑，则其值取0；若像素点颜色为白，则其值取1

3.2.3.2 提取图片数据特征

对已二值化处理的图片做以下特征提取：

  1. 建立edge矩阵提取图片边缘信息： $edge[i,j]$为$19*2$的矩阵，储存每一条碎片的边缘像素点信息。其中$i$表示碎片的编号，$j=0$表示左侧边缘像素点信息，$j=1$表示右侧边缘像素点信息。例如，$edge[5,0]$储存$004$号碎片左侧边缘像素点信息。
  2. 建立count矩阵提取图片边缘信息： 根据 $edge[i,j]$矩阵，我们可以计算得到一个 $19*2$ 的 $count[i,j]$矩阵，该矩阵储存每一条碎片边缘取值为 $0$ 的像素点的数量。例如，$count[0,0]=350$ 表示 $001$号碎片左侧边缘共有 $350$ 个黑色像素点。
  3. 位于最左端的纸片编号确定： 根据 count 矩阵，若 edge[i,0]=0，那么我们就认为该碎片在原始文件中处于最左端。

  4. 碎片行特征的提取： 由于原始文件文字行特征明显，因此我们需要提取碎片的行特征（其具体的原因见匹配度的定义）。首先确定碎片顶端取值为 $0$ 的像素点的位置，以此作水平线为上边界，依次向下取 $w$ 为行宽（这里取 $w=40 pixels$ 以保证能容纳每一个文字）直至下边缘，得到每条碎片的行数为$n_1, n_2,…, n_{19}$；然后取$n=\max (n_1, n_2,…,n_{19})$作为最终确定的行数，并以该条碎片的行化方式为标准对每一条碎片进行行化；最终每一条碎片被划分为 $28$ 行。每条碎纸片行特征大致列表如下：
